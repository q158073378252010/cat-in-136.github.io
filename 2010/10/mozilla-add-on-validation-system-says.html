Mozilla Add-on validation system says HTTP setting is generally not allowed.

Post date: 2010-10-03T23:22:00+09:00

Configuration Mania 1.12.2010091201 could not pass Mozilla Add-ons' review. Reviewer says HTTP Network settings are unsafe. On Mozilla Add-ons' validation description page, they say:

  • network.http.*
 Changing network preferences can cause subtle browsing bugs as well as problems with certain servers and ISPs. This is generally not allowed.

On the next release of Configuration Mania, I will either

  • remove all network setting so as to pass the review; or
  • give up to pass the review and Configuration Mania have not been reviewed.

Which way am I going?
